<h4>Non essere asociale.</h4><section><hr /><p>Durante lo studio di <a href="https://www.corsobitcoin.com/prodotti/libro-bitcoin-dalla-teoria-alla-pratica/" target="_blank" rel="noopener noreferrer" data-href="https://www.corsobitcoin.com/prodotti/libro-bitcoin-dalla-teoria-alla-pratica/"><strong>Bitcoin dalla teoria alla pratica</strong></a> e il <a href="https://www.udemy.com/course/bitcoin-blockchain-corso-completo-teoria-pratica-esempi-tutorial/?referralCode=AAC8EB895142D8301C13" target="_blank" rel="noopener noreferrer" data-href="https://www.udemy.com/course/bitcoin-blockchain-corso-completo-teoria-pratica-esempi-tutorial/?referralCode=AAC8EB895142D8301C13">relativo video-corso</a> abbiamo fatto un pò di esperimenti. Uno di questi è far comunicare due o più nodi sullo stesso computer.</p><p>Facendo riferimento all’articolo <a href="https://medium.com/@bitcoindallateoriallapratica/tutorial-creare-una-transazione-in-bitcoin-467c94fad754?source=---------2------------------" target="_blank" rel="noopener noreferrer" data-href="https://medium.com/@bitcoindallateoriallapratica/tutorial-creare-una-transazione-in-bitcoin-467c94fad754?source=---------2------------------">precedente</a>, utilizziamo un nuovo demone e cerchiamo di farli comunicare.</p><p>Sappiamo che di default le porte del demone di Bitcoin sono <strong>18444</strong> e la <strong>18443</strong>, per quanto riguarda l’ambiente di <strong>regtest</strong>.</p><pre>Ambiente: Porta - Porta RPC</pre><pre><code>Mainnet:8333 - 8332<br />Testnet:18333 - 18332<br />Regtest:18444 - 18443</code></pre><p>Quello che dobbiamo fare è creare un altro file di configurazione per il <strong>secondo nodo</strong> con un <strong>datadir</strong> differente e far comunicare i due nodi.</p><p>Passiamo alla action!</p></section><section><hr /><p>Questo è il mio percorso dove mi posizionerò</p><pre>$ pwd
/Users/barno/Documents/bizantino</pre><p>Al suo interno ho due cartelle, una un collegamento alla cartella di default di Bitcoin e l’altra è una cartella vuota pronta ad ospitare il secondo nodo.</p><pre>$ ls -l
total 0
lrwxr-xr-x  1 barno  staff   48 Jan 13 16:30 Bitcoin -&gt; /Users/barno/Library/Application Support/Bitcoin
drwxr-xr-x  4 barno  staff  128 Jan 13 17:23 <strong>Bitcoin_2</strong></pre><p>All’interno della cartella <strong>Bitcoin_2</strong> creo un file <strong>bitcoin_nodo2.conf </strong>e inserisco questa configurazione.</p><pre>datadir=/Users/barno/Documents/bizantino/Bitcoin_2
debug=1
regtest=1
# Options only for mainnet
[main]
# Options only for testnet
[test]
# Options only for regtest
[regtest]
port=28444
rpcport=28443
addnode=localhost:18444</pre><p>Come vedete sono stati cambiati la <strong>porta</strong> e la <strong>rpcport</strong>. <br />Senza il file di configurazione sarebbe stato molto più prolisso e soggetto ad errori, come si può vedere dall’esempio sotto.</p><pre>bitcoind -datadir=$PWD/regtest2 -regtest -debug=1 -rpcport=28443 -port=28444 -addnode=localhost:18444</pre><p>Per avere informazioni sui parametri da utilizzare, potete sempre utilizzare <strong>bitcoind -help</strong></p></section><section><hr /><figure><img alt="Due nodi Bitcoin sullo stesso computer" src="https://cdn-images-1.medium.com/max/800/1*FT9Rm6eZoiAv6702qzWMtA.png" data-image-id="1*FT9Rm6eZoiAv6702qzWMtA.png" data-width="808" data-height="134" /><p> </p><figcaption>Le porte utilizzabili.</figcaption></figure><ul><li><strong>-port</strong> Listen for connections on &lt;<strong>port</strong>&gt; (default: 8333, testnet: 18333, 18444 regtest)</li><li><strong>-rpcport</strong> Listen for JSON-RPC connections on &lt;port&gt; (default: 8332, testnet: 18332, regtest: 18443)</li><li><strong>-conf</strong>=&lt;file&gt; Specify configuration file. Relative paths will be prefixed by datadir location. (default: bitcoin.conf)</li><li><strong>-connect</strong>=&lt;ip&gt;<br />Connect only to the specified node; -noconnect disables automatic<br />connections (the rules for this peer are the same as for<br />-addnode). This option can be specified multiple times to connect<br />to multiple nodes.</li></ul></section><section><hr /><p>Passeremo al secondo nodo l’informazione necessaria per caricare il file di configurazione desiderato.</p><pre>-conf=/Users/barno/Documents/bizantino/Bitcoin_2/bitcoin_nodo2.conf </pre><p>avremo a disposizione due nodi sullo stesso computer.</p><blockquote><p>questo percorso sarà diverso nel vostro computer /Users/barno/Documents/bizantino/Bitcoin_2.</p></blockquote><p>Lanciamo quindi il demone di <em>default </em>(quello che abbiamo configurato nel precedente <a href="https://medium.com/@bitcoindallateoriallapratica/tutorial-creare-una-transazione-in-bitcoin-467c94fad754?source=---------2------------------" target="_blank" rel="noopener noreferrer" data-href="https://medium.com/@bitcoindallateoriallapratica/tutorial-creare-una-transazione-in-bitcoin-467c94fad754?source=---------2------------------">tutorial</a>) e il secondo demone</p><pre>$ bitcoind <br />$ bitcoind -conf=$PWD/bitcoin_nodo2.conf</pre><p><em>NB: utilizzo $PWD perchè sono nella stessa cartella del file bitcoin_nodo2.conf.</em><br />Per utilizzare il client del secondo demone devo passare sempre il parametro</p><pre><strong>-conf</strong></pre><p>Ehi vi vedete?</p><pre>$ bitcoin-cli -conf=$PWD/bitcoin_nodo2.conf getconnectioncount<br />1</pre><p>Sì si vedono, infatti il comando che conta le connessioni ha come risultato 1!<br />Adesso facciamo la grande prova, miniamo 101 blocchi sul nodo di default e verifichiamo se sul nodo 2 sono visibili.</p><p>Per prima cosa dobbiamo ottenere un address nel nodo di default.</p><pre>$ ADDR=`bitcoin-cli getnewaddress`</pre><p>Lo salvo direttamente nella variabile d’ambiente <strong>ADDR</strong> così da agevolare la scrittura dei comandi successivi.</p><p>Per visualizzare il valore di $ADDR, possiamo utilizzare <strong>echo</strong></p><pre>$ echo $ADDR
2N6yRwpVGdFsD1gBBhGTE8ijVmNwiVz6myZ</pre><p>Miniamo!</p><pre>$ bitcoin-cli generatetoaddress 101 $ADDR</pre><p>Otteniamo così un output molto corposo, ovvero 101 block header hash.<br />Per verificare che i blocchi si siano propagati correttamente, conto il numero di blocchi sul secondo nodo.</p><pre>$ bitcoin-cli -conf=$PWD/bitcoin_nodo2.conf getblockcount
101</pre><p>Si! Stanno parlando e sono perfettamente sincronizzati. <br />Dove sono salvate le relative blockchain?<br />Nel demone di default troviamo i blocchi nel percorso predefinito:</p><pre>/Users/barno/Library/Application Support/Bitcoin/regtest</pre><p>Il secondo demone utilizza la -datadir specificata nel file di conf.</p><pre>/Users/barno/Documents/bizantino/Bitcoin_2/regtest</pre><p>Potrebbe essere interessante iniziare a fare delle transazioni tra due nodi?<br />Direi di si!</p></section>
